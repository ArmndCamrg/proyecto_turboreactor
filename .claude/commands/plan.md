# /plan

Use this command to produce an implementation plan before making medium or large changes.

## Required output
- Objective
- Project type
- Scope
- Assumptions
- Risks
- Proposed design
- Files to create or modify
- Validation plan
- Deliverables
- Deferred items for phase 2

## Rules
- Prefer MVP-first design.
- Highlight security or infrastructure risks.
- Call out when a change touches high-risk paths.
- Do not execute destructive actions.
- Ask for approval before commands that modify environments or infra.
